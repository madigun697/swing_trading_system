# 미국 주식 스윙 투자 자동화 시스템 구축 계획서

# 1. 프로젝트 목표
미국 주식 시장을 대상으로 하는 스윙 트레이딩 자동화 시스템 구축.
목표:
- 미국 주식 데이터 자동 수집
- 정량 기반 종목 스크리닝
- 차트/거래량 기반 스윙 전략 수행
- 백테스트 및 성과 분석
- 뉴스/LLM 기반 정성 분석
- 실시간 손절/익절 모니터링
- 장기적으로 자동 매매 시스템으로 확장 가능하도록 설계

# 2. 기술 스택

### Python 환경

* Python 3.11+
* uv 기반 가상환경 및 dependency 관리

### 주요 프레임워크 및 라이브러리

| 영역	| 기술
|-----|----|
| Scheduler	| Apache Airflow
| DB	| PostgreSQL (Quant와 동일)
| Object Storage	| MinIO (Quant와 동일)
| Data Processing	| DBT
| 차트 데이터	| yfinance / Alpaca
| 뉴스 데이터	| Finnhub / NewsAPI
| 재무 데이터	| Financial Modeling Prep
| 백테스트	| vectorbt / backtrader
| API 서버	| FastAPI
| UI	| FastAPI와 Jinja2 템플릿 엔진을 활용한 Server-Side Rendering(SSR) 방식 (Quant와 동일)
| 시각화	| Plotly
| AI 분석	| OpenAI API

# 3. 단계별 구현 계획

## Phase 1. 데이터 수집 시스템 구축

### 목표

미국 주식 데이터를 자동으로 수집하고 저장.

### 유의사항

현재 quant_system를 통해 미국 주식 데이터가 수집되고 있습니다. 중복을 피하기 위해서 동일한 DB를 공유하고, 공유 가능한 데이터는 상호 공유하여 사용하도록 합니다.

## Phase 2. 스크리닝 및 타겟 종목 선정

### 목표

스윙 전략에 적합한 종목 자동 선정.

### 시장 상태 판단
- 1차 스크리닝: 비유동성/약세 종목 제거.
- 2차 스크리닝: 정량 점수화

### 차트 분석을 기반으로 한 스윙 투자 전략 구현
 - 눌림목 전략
 - 돌파 전략

### 포트폴리오 구성 규칙

```
최대 5종목
동일 섹터 최대 2개
종목당 계좌 리스크 1%
```

## Phase 3. 백테스트 기능 및 UI 구현

### 목표

전략 검증 및 성과 분석 시스템 구축.

### 핵심 원칙

#### Look-Ahead Bias 방지
```
과거 특정 날짜 기준으로
그 날짜까지의 데이터만 사용
```

### 백테스트 흐름
```
과거 날짜 반복
→ 해당 날짜 기준 스크리닝
→ Top 종목 선정
→ 다음날 시가 진입
→ 손절/익절 체크
→ 거래 기록 저장
→ 성과 분석
```

### 백테스트 엔진

#### 추천

##### vectorbt

* 빠른 다중 전략 테스트

##### backtrader

* 실전 주문 구조 구현 가능

### 백테스트 기능

| 기능 | 설명 |
|-----|----|
| 거래비용 | 수수료/슬리피지 |
| 손절 | ATR/전저점/고정% |
| 익절 | 1R/2R/trailing |
| 포지션 사이징 | 계좌 리스크 기준
| 리밸런싱 | 주간

### 성과 지표
```
CAGR
MDD
Sharpe Ratio
Win Rate
Profit Factor
Expectancy
월별 수익률
종목별 기여도
```

### UI 구성

#### 화면 구성

```
데이터 상태
스크리닝 결과
종목별 차트
백테스트 설정
백테스트 결과
거래 로그
성과 분석
```

## Phase 4. 뉴스/LLM 정성 분석

### 목표

정량 분석 결과에 뉴스/이벤트 기반 정성 분석 추가.

### 수집 데이터

- 데이터	API
- 뉴스	Finnhub
- 실적 발표	FMP
- SEC Filing	SEC EDGAR
- 애널리스트 의견	FMP

### 중요한 원칙

LLM은 보조 역할

LLM이 직접 매수/매도를 결정하지 않음.

LLM 역할:

* 뉴스 요약
* 감성 분석
* 리스크 설명
* 이벤트 분류

## Phase 5. 실시간 손절/익절 모니터링

### 목표

보유 종목에 대한 리스크 관리 자동화.

### 규칙 예시

손절
```
전저점 이탈
20MA 이탈
-1R 도달
```

익절
```
+2R 절반 매도
Trailing Stop
```

### 추천 구조

별도 서비스 운영

Airflow 대신: FastAPI Worker 또는 Async Python Service 추천

### 실시간 모니터링 흐름

가격 수집
→ 손절/익절 조건 계산
→ 주문 신호 생성
→ Paper Trading 또는 실거래 API 실행
